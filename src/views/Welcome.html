Welcome to the account management backend

User Credentials:

username: username, password: password (-> role: user)

username: admin, password: admin (-> role: admin)

The User Schema:

        token: { type: String, required: true },
        title: { type: String }, // maybe predefined and select via dropdown
        gender: { type: String }, // select via dropdown?
        firstname: { type: String, required: true, trim: true },
        lastname: { type: String, required: true, trim: true },
        username: { // needs to be trimmed in frontend?
            type: String, unique: true, required: true, trim: true,
        },
        email: {
            type: String, unique: true, required: true, trim: true,
        },
        password: { type: String, required: true },
        organisation: { type: String, trim: true },
        address: { type: String, trim: true },
        city: { type: String }, // maybe also predefined via npm module
        country: { type: String }, // maybe also predefined via npm module
        zipCode: { type: Number },
        fieldOfActivity: { type: String, required: true, trim: true },
        researchInterest: { type: Array, required: true }, // predefined in enums
        role: { type: String, enum: ['admin', 'user'], default: 'user' },
        securityQuestion: { type: String, required: true }, // predefined in enums
        securityAnswer: { type: String, required: true, trim: true },
        eventbasedRole: [
            {
            role: { type: String },
            event: { type: Number }, // contains id of event
            },
        ],

How to use our service:


Collections

  • GET /securityQuestions - to get all possible security questions
  • GET /researchInterests - to get all possible research Interests

User

  • POST /login - needs redirectUrl on where to go as parameter and username + password as body - on success the user is stored in the cookies and the redirectURL is used
  • POST /register - needs a userobject as body (minumum requirements: firstname, lastname, username, email, password, fieldOfActivity, researchInterest, securityQuestion, securityAnswer)
  • GET /uniqueUsername - needs username as parameter and checks if db already contains that value
  • GET /uniqueEmail - needs email as parameter and checks if db already contains that value
  • GET /checkSecurityAnswer - needs user id and his input answer and check if the saved answer matches
